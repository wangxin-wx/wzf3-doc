部署 ​

启动 ​

当 build 完成后，在项目根目录会生成 .dist 目录，可以使用 pm2 守护进程或 bash 临时启动预览。

bash
# pm2
pm2 start ./.dist/ecosystem.config.cjs
# bash
bash ./dist/start.sh

环境变量 ​

尽管是用 wzf3 build --config <config> 根据不同 config 构建生产包，当部署时，运行环境仍可能存在变化，此时可以配置环境变量修改配置。

例如，当构建时，配置如下：

ts
export const wzf: ModuleOptions = {
  ...,
  encrypt: false,
  proxyTimeout: 1000 * 60 * 10,
  logger: {
    showReq: true,
    showRes: true,
  },
}

在部署环境需要修改 encrypt、proxyTimeout和logger.showReq，可添加环境变量：

js
// .dist/ecosystem.config.cjs
const envs={
  "PWD": __dirname,
  WZF_PUBLIC_WZF_ENCRYPT:true,
  WZF_PUBLIC_WZF_PROXY_TIMEOUT:1000,
  WZF_PUBLIC_WZF_LOGGER_SHOW_REQ:false,
  ...,
}

变量命名规则：以 WZF_PUBLIC_WZF_ 作为前缀，遇到大写或者子属性时，添加下划线
